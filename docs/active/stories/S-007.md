## Presentation Manager

An admin panel for managing and running synchronized presentations. Built for a solo dev demoing over a video call — instead of screen sharing, viewers load the actual components in their own browser and the presenter controls playback from a dashboard. Everyone sees the same thing at the same time, at full fidelity.

## Concept

A **presentation** is a named sequence of steps. Each step maps to a segment of the 0-to-1 progress range from S-006. When the presenter hits "next," the progress value jumps to the next step's position, and every viewer's instance advances in sync.

For example, a 5-step presentation maps to progress values `[0.0, 0.25, 0.5, 0.75, 1.0]`. The presenter sees "Step 3 of 5" with prev/next controls. Viewers see whatever their component renders at progress `0.5`.

The admin panel lets you:
- Create and configure presentations (name, number of steps, optional step labels)
- Launch a presentation — creates a sync session and produces a shareable viewer URL
- Control playback — next/prev step, jump to step, or fine scrub with slider
- See how many viewers are connected

## Why

S-006 proved the sync primitive works. This story wraps it in a usable workflow: create a presentation, share a link, present. It's the minimum viable "present components over the web instead of screen sharing" experience. Once this works, plugging in actual React animation components is just a matter of rendering them at the given progress value.

## Architecture Notes

- **presentations** collection stores presentation metadata (name, step_count, step_labels, active_session relation)
- Going "live" creates a `sync_session` (from S-006) and links it to the presentation
- Presenter controls use the same `POST /api/sync/:id/progress` endpoint but with step-aware UI
- Viewer page extends the S-006 viewer — same realtime subscription, but with step context (step number, label)
- PocketBase JS SDK's realtime subscriptions handle all the live update plumbing
