## Data Model and Collections

Define PocketBase collections and seed data for a "link bookmarking" practice app. Users can save links with tags, and the app tracks view counts.

## Goal

PocketBase collections exist for:
- **links**: url, title, description, tags (relation), view_count, created_by (relation to users)
- **tags**: name, slug (unique)

Seed migration populates sample data for local development.

## Why

Gives the Astro frontend and custom Go routes concrete data to work with. Exercises PocketBase's migration system, collection API, and relation fields.
