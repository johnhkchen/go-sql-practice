## Project Scaffold

Stand up the foundational project structure: a Go binary that embeds PocketBase as a library, exposes custom API routes alongside PocketBase's built-in ones, and serves an Astro frontend from the embedded filesystem.

## Goal

A single `go build` produces one binary that:
- Starts PocketBase with its admin UI and default REST API
- Registers at least one custom Go route (`GET /api/health`)
- Serves the Astro frontend's static build output on the root path (`/`)

## Why First

Everything else depends on the project compiling and running. This story establishes the module layout, build pipeline, and development workflow before any feature work begins.
