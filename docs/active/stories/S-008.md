## Viewer Waiting Room

A polished Astro landing page that viewers see when they open the viewer URL before (or between) a live presentation. Instead of a bare "not live" message, they get a branded waiting room that auto-transitions into the live view the moment the presenter goes live.

## Goal

When a viewer opens `/watch/:id`:
- If the presentation is **not live**: show a waiting room with the presentation name, a "waiting for presenter" state, and a subtle animation so it doesn't feel broken
- If the presentation **goes live** while they're waiting: seamlessly transition into the live presentation view — no page reload
- If the presentation is **already live**: skip the waiting room and go straight to the live view

## Why

This is what the audience sees first. A good waiting room sets expectations ("you're in the right place, the presenter will start soon") and avoids confusion. The auto-transition via PocketBase realtime means viewers don't need to refresh — they just wait and the presentation appears.

## Architecture Notes

- Subscribe to the `presentations` record via PocketBase realtime — when `active_session` changes from null to a session ID, trigger the transition
- The waiting room and live view can be the same Astro page with different states (no route change needed)
- Keep the waiting room lightweight — it's Astro static HTML with one client island for the realtime subscription
