## Astro Frontend

Build the Astro static frontend that consumes the PocketBase API (both built-in and custom routes) and renders the link bookmarking UI.

## Goal

Astro pages and components:
- **Home page** (`/`): lists links with tag pills, search bar, sort options
- **Link detail** (`/links/:id`): shows full link info, increments view count on load
- **Stats page** (`/stats`): displays aggregate stats from the custom endpoint
- **Tag page** (`/tags/:slug`): shows all links for a given tag

Uses Astro's static site generation with client-side fetch for dynamic data (search, view counts).

## Why

Practices Astro's component model, client-side islands for interactivity, and wiring a frontend to a PocketBase backend. The mix of static pages and dynamic data exercises both SSG and client-side patterns.
