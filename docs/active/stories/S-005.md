## Testing and CI

Add test coverage for the custom Go routes and set up a basic CI pipeline that builds the full binary and runs tests.

## Goal

- Go integration tests that start an in-memory PocketBase instance, seed test data, and exercise each custom endpoint
- A Makefile (or Taskfile) that orchestrates: `build-frontend`, `embed`, `test`, `build`
- A GitHub Actions workflow that runs the full build + test pipeline

## Why

Validates that the Go + PocketBase + Astro build pipeline works end-to-end in automation, and that the custom SQL-heavy routes have regression protection.
