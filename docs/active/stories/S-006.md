## Synchronized Playback Bar

A minimal "shared progress" primitive: one admin controls a 0-to-1 float, and any number of viewers see it update in realtime. This is the foundation for synchronized React animation playback across devices — instead of streaming low-fidelity video, everyone loads the same component and a single progress value keeps them in lockstep.

## Goal

For this first pass, keep it simple:
- An admin creates a **session** and gets two URLs: an admin control URL and a shareable viewer URL
- The **admin page** shows a slider (0 to 1) that updates the session's progress value
- The **viewer page** renders a progress bar that live-updates via PocketBase's realtime subscriptions (SSE)
- No auth beyond a simple admin token in the URL. No React yet — just a native HTML progress bar to prove the sync works

## Why

Proves the realtime sync pattern end-to-end with minimal moving parts. Once this works, swapping the progress bar for a React animation component driven by the same 0-to-1 value is a straightforward next step.

## Architecture Notes

PocketBase provides realtime subscriptions out of the box — clients subscribe to record changes via SSE and get pushed updates. The admin writes to a `sync_sessions` record's `progress` field, PocketBase broadcasts the change, and all subscribed viewers receive it. No custom WebSocket server needed.
