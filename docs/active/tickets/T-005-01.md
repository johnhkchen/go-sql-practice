## Context

Write Go integration tests for the three custom API endpoints. PocketBase supports testing by creating an in-memory app instance, so tests can exercise the full HTTP handler stack without needing an external process.

## Acceptance Criteria

- Test file(s) in a `routes/` or `handlers/` test package
- Test helper that creates an in-memory PocketBase instance with collections and seed data
- Tests for search endpoint: basic query, tag filter, combined filters, empty results, pagination
- Tests for view count endpoint: successful increment, nonexistent ID returns 404, count actually increases
- Tests for stats endpoint: returns expected shape, counts match seed data
- `go test ./...` passes with all tests green
- Tests do not require a running PocketBase server (fully in-process)
