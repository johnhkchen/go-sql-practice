## Context

Implement `POST /api/links/:id/view` to atomically increment a link's view count. This exercises atomic SQL updates and PocketBase's record manipulation from Go.

## Acceptance Criteria

- `POST /api/links/:id/view` increments `view_count` by 1 and returns the updated record
- The increment is atomic (uses SQL `UPDATE ... SET view_count = view_count + 1`, not read-modify-write)
- Returns 404 with a JSON error if the link ID doesn't exist
- Response includes the full link record with updated `view_count`
- No authentication required (public endpoint for this practice app)
- Concurrent requests don't lose increments (verified by test or reasoning about the SQL)
