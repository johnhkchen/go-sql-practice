## Context

Pin all project tooling to their latest stable versions via flox before any code is written. Flox provides reproducible, declarative package management so every developer and CI run uses identical tool versions.

## Target Versions (as of 2026-02-21)

| Tool       | Version  | Flox package    | Notes                                    |
|------------|----------|-----------------|------------------------------------------|
| Go         | 1.26.0   | `go_1_26`       | Released 2026-02-10                      |
| Node.js    | 24.13.0  | `nodejs_24`     | Required by Astro 5                      |
| GNU Make   | latest   | `gnumake`       | Build orchestration                      |
| PocketBase | v0.36.5  | *(Go dep only)* | Go SDK, not a CLI package                |
| Astro      | 5.17.3   | *(npm dep only)*| Stable; v6 is beta                       |

## Acceptance Criteria

- [x] `.flox/env/manifest.toml` exists with `go_1_26@1.26.0`, `nodejs_24@24.13.0`, `gnumake`
- [x] `flox activate` puts `go`, `node`, `npm`, and `make` on PATH at the pinned versions
- [ ] `go.mod` (when created in T-001-01) specifies `go 1.26`
- [ ] `go.mod` requires `github.com/pocketbase/pocketbase v0.36.5`
- [ ] `frontend/package.json` (when created in T-001-03) pins `astro` to `^5.17.3`
- [ ] `.github/workflows/ci.yml` (when created in T-005-03) uses `flox activate` or matches pinned versions
- [ ] All version choices are documented in this ticket for future reference
