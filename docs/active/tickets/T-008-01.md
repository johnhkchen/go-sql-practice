## Context

Build the waiting room landing page that viewers see at `/watch/:id` when the presentation isn't live. This is the first thing the audience sees — it needs to look polished and clearly communicate "you're in the right place, the presenter hasn't started yet."

## Acceptance Criteria

- `/watch/[id]` fetches the presentation record on load
- When `active_session` is null (not live), renders the waiting room:
  - Presentation name as a heading
  - "Waiting for presenter to start..." status message
  - A subtle CSS animation (pulse, breathing dot, or loading shimmer) so the page feels alive
  - Clean, centered layout — minimal and professional
- When `active_session` is set (already live), skips waiting room and renders the live view directly
- Shows "Presentation not found" for invalid IDs
- Works on mobile — responsive, readable on phone screens
- No JavaScript required for the initial waiting room render (progressive enhancement)
- Uses the BaseLayout from T-004-01
