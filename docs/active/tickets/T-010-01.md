## Context

Multiple files contain dead code, duplicate implementations, and unused exports left over from early development. This dead code adds confusion, increases maintenance burden, and in the case of `links_search_simple.go` duplicates functionality already in `links_search.go`.

## Changes

### Delete `routes/links_search_simple.go` entirely
- Duplicate of `links_search.go`, registered as `/api/links/search-simple` which isn't in spec
- Remove its `registerLinksSearchSimple(e)` call from `routes/routes.go`

### Delete dead functions in `routes/presentations.go`
- `validateToken()` (line 174) — unused, `sync_sessions.go` has its own `validateSyncToken()`
- `stepToProgress()` (line 106) — unused inverse of `progressToStep()`
- Remove stale comment `// Placeholder handler functions - to be implemented in subsequent steps` (line 181)
- Remove commented-out import `// "encoding/json"  // TODO: Re-enable if JSON operations are added`

### Delete dead test scaffolding in `routes/routes_test.go`
- `TestData` struct (line 17) — never instantiated
- `assertErrorResponse()` function (line 157) — never called
- Note: `makeRequest()`, `echo/v5`, and `httptest` ARE actively used by `TestMakeRequest_RealExecution` — do not remove

### Remove `FrontendExists()` from `internal/frontend/embed.go`
- Exported function, never called outside the package
- Update `routes/routes.go` to call `registerStatic(e)` unconditionally (the function itself already handles errors gracefully via early return)

### Clean up `routes/static.go`
- Remove the unreachable fallback block (lines 74-79: `ev.Response.Write([]byte("file serving not supported"))`) — if `file.(io.ReadSeeker)` fails, `ev.Next()` is more appropriate than returning a raw string
- Keep the `spaFS` struct and overall handler structure

### Run `go mod tidy`
- Run `go mod tidy` to clean up any unused transitive dependencies
- Note: `labstack/echo/v5` IS used by `routes_test.go` (`makeRequest` uses `echo.New()`) — it will not be removed

## Acceptance Criteria

- `links_search_simple.go` is deleted
- No references to `registerLinksSearchSimple` or `FrontendExists` remain
- `go vet ./...` passes
- `go build .` compiles without errors
- `go mod tidy` results in no diff
- No commented-out imports remain in modified files
