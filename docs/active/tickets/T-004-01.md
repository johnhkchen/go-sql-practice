## Context

Create the base Astro layout and navigation component. All pages share a common shell (header, nav, footer) so this needs to exist before individual pages are built.

## Acceptance Criteria

- `BaseLayout.astro` component provides HTML shell (head, body, slot for content)
- Navigation component with links to: Home (`/`), Stats (`/stats`)
- Basic CSS (can use Astro's built-in scoped styles or a minimal utility approach)
- Layout is responsive — usable on mobile widths
- All pages created in subsequent tickets use this layout
