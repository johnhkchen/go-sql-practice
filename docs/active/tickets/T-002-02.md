## Context

Create a seed migration that populates the database with sample data for local development. Having realistic data from the start makes frontend development and API testing faster. The seed should be idempotent — safe to run multiple times without duplicating records.

## Acceptance Criteria

- Seed migration runs after the collection-creation migration
- Creates at least 5 tags (e.g., "golang", "javascript", "database", "devops", "frontend")
- Creates at least 10 links with varied tags, titles, and descriptions
- Links have realistic URLs (can be example.com or real documentation links)
- Some links have multiple tags to exercise the relation field
- `view_count` values vary (0 to ~50) so stats queries have interesting data
- Running the app fresh and checking `/api/collections/links/records` returns the seeded links
