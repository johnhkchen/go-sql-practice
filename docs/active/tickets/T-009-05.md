## Context

`routes/routes_test.go` has a `makeRequest()` function that returns hardcoded mock responses instead of making real HTTP requests. The `httptest` import is commented out. Tests always pass regardless of whether endpoints actually work. This defeats the purpose of integration tests.

## Acceptance Criteria

- `makeRequest()` uses `httptest` to make real HTTP requests against a PocketBase test instance
- Or, if PocketBase v0.36.5 doesn't support `httptest`, use `app.OnServe` with a test router
- Tests in `links_search_test.go` actually exercise the real search endpoint and validate response data
- Add basic tests for: view count increment, stats endpoint, sync create, sync progress update
- `go test ./...` passes with real endpoint tests (not mocked)
- At least one test fails if an endpoint is broken (prove tests aren't vacuous)
