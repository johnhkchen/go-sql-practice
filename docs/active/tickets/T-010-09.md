## Context

T-010-03 created `apiFetch<T>()` in `lib/api.ts` but no page uses it — every SSR page still does manual `fetch()` + `AbortController` + `setTimeout`. T-010-06 extracted scripts to `.ts` files but several still read `(window as any).PUBLIC_API_URL` instead of importing `API_BASE` from `lib/api.ts`. The extracted files also split across `scripts/` and `lib/` inconsistently.

## Changes

### Either use `apiFetch<T>()` or delete it
Option A (preferred): Refactor SSR pages to use `apiFetch<T>()`, replacing the duplicated fetch+abort+timeout pattern in:
- `index.astro` (~15 lines of fetch boilerplate)
- `tags/[slug].astro`
- `links/[id].astro`
- `present/index.astro`
- `present/[id].astro`
- `sync/[id]/control.astro`
- `watch/[id].astro`

Option B: Delete `apiFetch<T>()` and `ApiError` from `lib/api.ts` if the abstraction doesn't fit the actual usage patterns.

### Replace `(window as any).PUBLIC_API_URL` with `API_BASE` import
Files that read API_BASE from `window` instead of importing from `lib/api.ts`:
- `frontend/src/lib/statsController.ts` (lines 13-14)
- `frontend/src/lib/searchEnhancer.ts` (line 40)
- `frontend/src/components/PresenterController.astro` (line 782)

Note: Client-side `<script>` tags in Astro may not have access to `import.meta.env` at runtime. If `API_BASE` from `lib/api.ts` doesn't work in client scripts, use `data-*` attributes on DOM elements to pass the value from the SSR frontmatter instead of `window.*` globals.

### Remove `window.*` global assignments for data passing
- `present/[id].astro` lines 275-278: sets `window.presenterSessionId`, `window.presenterAdminToken`, `window.presenterData`, `window.PUBLIC_API_URL` — use `define:vars` or `data-*` attributes instead
- `sync/[id]/control.astro` line 864: `window.syncController = new SyncController(...)` — remove global assignment, keep local instantiation
- `sync/[id].astro` line 137: `window.syncViewer = new SyncViewer(...)` — same
- `watch/[id].astro` line 950: `window.presentationAutoViewer = new PresentationAutoViewer(...)` — same
- `components/StatsSummary.astro` line 68: `(window as any).refreshStats = () => { ... }` — same

### Consolidate extraction directory
All 6 extracted `.ts` files should be in one directory. Currently:
- `frontend/src/scripts/syncViewer.ts` (the outlier)
- `frontend/src/lib/syncController.ts`
- `frontend/src/lib/presentationViewer.ts`
- `frontend/src/lib/presenterController.ts`
- `frontend/src/lib/statsController.ts`
- `frontend/src/lib/searchEnhancer.ts`

Move `syncViewer.ts` to `lib/` (or move all to `scripts/` — pick one) and update the import in `sync/[id].astro`.

## Acceptance Criteria

- `apiFetch<T>()` is either used by at least 3 pages or deleted
- No `(window as any).PUBLIC_API_URL` patterns remain
- No `window.presenterSessionId` / `window.presenterData` / etc. assignments
- All extracted `.ts` controller files are in the same directory
- `cd frontend && npm run build` succeeds
