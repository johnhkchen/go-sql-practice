## Context

The frontend has accumulated HTML semantics issues, dead CSS, accessibility gaps, hardcoded values, and ~40 production `console.log` statements across 6+ files. These don't break functionality but hurt code quality and accessibility compliance.

## Changes

### Fix Astro config
- `astro.config.mjs`: change `output: 'static'` to `output: 'hybrid'` — the app uses `export const prerender = false` on most pages, which requires hybrid or server mode. The `static` setting contradicts actual SSR usage.

### Create shared `<ErrorState>` component
- Create `frontend/src/components/ErrorState.astro` with props: `title`, `message`, `showRetry`, `backLink`, `backLabel`
- Replace duplicated error HTML/CSS blocks in: `index.astro`, `tags/[slug].astro`, `links/[id].astro`, `present/index.astro`, `present/[id].astro`
- Each page currently has ~50 lines of identical error state markup and ~80 lines of identical error CSS

### Fix nested `<main>` elements
- `BaseLayout.astro` already wraps `<slot />` in `<main>` (line 100)
- Pages that add their own `<main>` create invalid nested `<main>` elements:
  - `index.astro`: change `<main class="home-page-container">` to `<div class="home-page-container">`
  - `tags/[slug].astro`: change `<main class="tag-page-container">` to `<div class="tag-page-container">`
  - `present/[id].astro`: change `<main class="presenter-control-page">` to `<div class="presenter-control-page">`
  - `PresenterController.astro`: change `<main class="controller-main">` to `<div class="controller-main">`

### Remove `role="main"` from `LinksList.astro`
- Line 98: `<div class="links-grid" role="main">` — a `<div>` should not have `role="main"`, especially inside an actual `<main>` element
- Remove the `role="main"` attribute

### Fix tag links in `links/[id].astro`
- Line 114: tag links use `/search?tag=${tag}` — the correct route is `/tags/${tag}`
- Line 377: same issue in the script block tag resolution

### Fix `Navigation.astro` accessibility
- Add `aria-current="page"` to the active nav link (compare `Astro.url.pathname` to link hrefs)
- Replace checkbox hamburger hack with proper `<button>` element with `aria-expanded` and `aria-controls` attributes
- Add `aria-label="Main navigation"` to the `<nav>` element

### Fix copyright year in `BaseLayout.astro`
- Line 105: hardcoded `&copy; 2024` — change to dynamic `{new Date().getFullYear()}`

### Remove dead CSS
- `tags/[slug].astro`: `.loading-skeleton` class and `@keyframes loading` animation (lines 350-362) — never used in the template
- `sync/[id]/control.astro`: `.temp-debug` CSS rules (lines 1429-1452) — debug styling not used in template

### Remove production `console.log` statements
- ~40 `console.log` calls across 6 files (see `grep -r "console.log" frontend/src/`)
- Remove all non-error logging: `console.log('Home page loaded')`, `console.log('Page type:')`, `console.log('Tag page loaded')`, `console.log('Viewing tag:')`, `console.log('Client-side search enhancement loaded')`, `console.log('SyncController initialized')`, etc.
- Keep `console.error` calls for actual error handling
- Replace debug-useful logs with no-ops or remove entirely

### Fix relative API URLs
- `GoLive.astro` line 38: `fetch('/api/presentations/...')` uses relative URL — should use `API_BASE` from `lib/api.ts` (or the shared `apiFetch`)
- Check `PresenterController.astro` and `present/index.astro` for same issue

### Fix `PresenterController.astro` data passing
- If using `window.*` globals to pass data from Astro to client script, switch to `define:vars` directive or `data-*` attributes

### Add semantic color tokens to `:root` in `BaseLayout.astro`
- Add `--color-success`, `--color-error`, `--color-secondary`, `--border-radius` CSS custom properties
- Replace hardcoded color values in error/success states across components

## Acceptance Criteria

- No nested `<main>` elements (validate HTML structure)
- No `role="main"` on non-main elements
- Tag links navigate to `/tags/` routes, not `/search?tag=`
- Navigation has `aria-current="page"` on active link
- Copyright year is dynamic
- No `.loading-skeleton` CSS in `tags/[slug].astro`
- `grep -r "console.log" frontend/src/` returns 0 results (or only justified debug lines)
- `GoLive.astro` uses `API_BASE` not relative `/api/` URL
- `cd frontend && npm run build` succeeds
