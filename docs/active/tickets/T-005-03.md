## Context

Set up a GitHub Actions workflow that runs the full build and test pipeline on every push and PR. Validates that the project builds cleanly in a fresh environment.

## Acceptance Criteria

- `.github/workflows/ci.yml` defines the CI pipeline
- Workflow triggers on push to `main` and on pull requests
- Steps: checkout, set up Go, set up Node.js, install frontend deps, build frontend, run Go tests, build Go binary
- Go version pinned to `1.26`, Node version pinned to `24`
- Workflow caches Go modules and npm dependencies for faster runs
- Pipeline fails if tests fail or if the binary doesn't compile
