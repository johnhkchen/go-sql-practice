## Context

Add realtime detection so the waiting room automatically transitions to the live presentation view when the presenter goes live. Viewers should never need to refresh — the page watches for the presentation's `active_session` to change and seamlessly swaps in the live view.

## Acceptance Criteria

- An Astro client island (`client:load`) subscribes to the `presentations` record via PocketBase realtime
- When `active_session` changes from null to a session ID:
  - Transitions from waiting room state to live presentation view (same page, state change — no full reload)
  - Begins subscribing to the `sync_sessions` record for progress updates (same as T-007-04 viewer)
  - Shows a brief "Presentation starting..." transition state
- When `active_session` changes from a session ID to null (presenter stops):
  - Transitions back to the waiting room state
  - Shows "Presentation ended" with option to keep waiting for next session
- Connection status indicator visible during waiting room (connected / reconnecting)
- Handles edge cases:
  - Viewer opens page while presentation is mid-session (join in progress — fetch current progress and render)
  - Network disconnect and reconnect (resubscribe and sync to current state)
- Transition is visually smooth (fade or slide, not a jarring swap)
