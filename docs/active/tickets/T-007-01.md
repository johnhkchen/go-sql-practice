## Context

Create a PocketBase collection for presentations. A presentation is a named sequence of steps, each mapping to a segment of the 0-to-1 progress range. When a presentation goes live, it links to a sync_session from S-006.

## Acceptance Criteria

- Migration creates a `presentations` collection with fields:
  - `name` (text, required) — human-readable presentation name
  - `step_count` (number, required, min 1) — total number of steps
  - `step_labels` (json, optional) — array of strings labeling each step (length should match step_count)
  - `active_session` (relation to sync_sessions, optional) — the currently live sync session, null when not presenting
  - `created_by` (relation to users, optional) — owner
- Collection runs alongside existing migrations without conflict
- A presentation with `step_count: 5` implies progress breakpoints at `[0.0, 0.25, 0.5, 0.75, 1.0]`
- API rules allow authenticated users to create/edit their own presentations, anyone to view
