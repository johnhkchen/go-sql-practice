## Context

`frontend/src/components/StatsSummary.astro` line 141 hardcodes the API URL to `http://127.0.0.1:8094`. Other components may have similar issues. The frontend should use a configurable base URL or relative paths (since it's served from the same origin).

## Acceptance Criteria

- `StatsSummary.astro` uses a relative URL (`/api/stats`) or an environment variable (`PUBLIC_API_URL`), not a hardcoded host:port
- Audit all other `.astro` files for hardcoded API URLs and fix any found
- Frontend works when served from the Go binary (same-origin, relative paths)
- Frontend works in dev mode with Astro's dev server (configurable proxy or env var)
