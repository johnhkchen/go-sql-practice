## Context

The presenter's dashboard — the main control surface for managing and running presentations. A solo dev opens this before a call, picks a presentation, goes live, shares the viewer link, and controls step progression during the demo.

## Acceptance Criteria

- Dashboard page at `/present` listing all presentations (fetched from PocketBase built-in API)
- Each presentation shows: name, step count, live/offline status
- "New Presentation" form: name, step count, optional step labels (comma-separated)
- "Go Live" button on a presentation calls `POST /api/presentations/:id/live`
- Once live, redirects to presenter control view at `/present/[id]`
- Presenter control view shows:
  - Current step indicator: "Step 3 of 5 — [step label]"
  - **Prev / Next** buttons for step-by-step navigation
  - Step jump buttons (clickable step indicators, like dots or numbered pills)
  - Fine-control slider for scrubbing between steps (maps to 0-1 progress)
  - Shareable viewer URL with copy button
  - "Stop Presenting" button that calls the stop endpoint and returns to dashboard
- Step navigation calls `POST /api/sync/:id/progress` with the computed progress value
- Prev button disabled at step 0, next disabled at last step
- Uses Astro client islands for all interactive controls
