## Context

Build tooling, CI, and project configuration have gaps: missing `.gitignore` entries allow test artifacts into version control, stray directories exist from earlier development, CI is missing lint/vet steps, and `CLAUDE.md` has a placeholder description.

## Changes

### `.gitignore` updates
- Add `*.test` to prevent Go test binaries from being committed (e.g., `routes.test` currently tracked)
- Add `routes/pb_data/` to prevent test database artifacts

### Delete stray directories
- Delete `frontend/frontend/src/` — this is a duplicate directory created by a build script error
- Delete `routes/pb_data/` — test artifact that shouldn't exist in routes directory

### Fix embed path in Makefile
- The `backend` target builds Go but doesn't ensure `frontend/dist` is copied to `internal/frontend/frontend/dist/` where `embed.go` expects it
- Add a step after `frontend` target: `cp -r $(DIST_DIR) internal/frontend/frontend/dist`
- Or adjust the build to symlink / ensure the embed path matches

### Fix `make help` target mismatch
- Line 74: `help` lists `validate` but the actual target is `validate-build`
- Change `"  validate    - Validate build artifacts"` to `"  validate-build - Validate build artifacts"`

### Add `lint` and `vet` Makefile targets
- `vet`: `go vet ./...`
- `lint`: `gofmt -l .` (list files that need formatting; fail if any output)
- Add both to `.PHONY`

### Update CI workflow
- Update `actions/setup-go@v4` to `actions/setup-go@v5`
- Add `go vet ./...` step after build
- Add `gofmt` check step (fail if any files need formatting)
- Add `npm audit --audit-level=high` step after frontend build
- Optionally add `govulncheck ./...` step (if available in CI Go version)

### Create `frontend/.env.example`
- Contents: `PUBLIC_API_URL=http://127.0.0.1:8090`
- Documents the required environment variable for frontend development

### Align port numbers
- Verify that all references to the API port are consistent: Makefile (`SERVER_PORT`), frontend env default, Go code fallbacks should all use `8090`
- Currently aligned — document this as the canonical port

### Update `CLAUDE.md`
- Fill in the project description placeholder: "Link bookmarks app with Go/PocketBase backend and Astro frontend"
- Add any relevant build/run instructions

### Fix internal error detail leakage
- Several Go handlers return raw error details in API responses (e.g., `fmt.Sprintf("Query failed: %v", err)`)
- Replace with generic user-facing messages and log the detail server-side
- Files: `links_search_simple.go` (deleted by T-010-01), check remaining handlers in `links_search.go`, `links_view.go`, `stats.go`

## Acceptance Criteria

- `*.test` files excluded by `.gitignore`
- `frontend/frontend/src/` directory deleted
- `routes/pb_data/` directory deleted
- `make help` output matches actual target names
- `make lint` and `make vet` targets exist and work
- CI workflow uses `setup-go@v5`
- CI includes vet and format check steps
- `frontend/.env.example` exists with documented `PUBLIC_API_URL`
- `CLAUDE.md` has a real project description
- No internal error details in API response bodies
