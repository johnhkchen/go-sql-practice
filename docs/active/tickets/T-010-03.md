## Context

Every SSR page in the Astro frontend independently defines `API_BASE`, `FETCH_TIMEOUT`, `AbortController` timeout patterns, and identical interface types (`LinkItem`, `SearchResponse`, etc.). This copy-paste pattern makes changes error-prone and inflates file sizes.

## Changes

### Create `frontend/src/types/api.ts`
- Canonical `LinkItem` interface (consolidating copies in `LinkCard.astro`, `SearchInterface.astro` script block, `tags/[slug].astro`, `links/[id].astro`, `index.astro`)
- `SearchResponse` interface
- `PocketBaseResponse` interface
- `StatsData` and `StatsState` interfaces (currently in `StatsSummary.astro`)
- `Presentation` and `PresentationStatus` interfaces (currently in `present/index.astro`)
- Export all types for use in both frontmatter and client scripts

### Create `frontend/src/lib/api.ts`
- Shared `API_BASE` constant: `import.meta.env.PUBLIC_API_URL || 'http://localhost:8090'`
- Shared `FETCH_TIMEOUT` constant (5000ms)
- Generic `apiFetch<T>(url: string, opts?: RequestInit): Promise<T>` helper that:
  - Creates `AbortController` with timeout
  - Sets `Accept: application/json` header
  - Throws typed errors for timeout/network/HTTP failures
  - Returns parsed JSON

### Refactor SSR pages to use shared utilities
Update the following pages to import from `types/api.ts` and `lib/api.ts` instead of inline definitions:
- `index.astro` — remove inline `PocketBaseResponse`, `SearchResponse`, `LinkItem`, `API_BASE`, `FETCH_TIMEOUT`, AbortController pattern
- `tags/[slug].astro` — remove inline `LinkItem`, `SearchResponse`, `API_BASE`, `FETCH_TIMEOUT`
- `links/[id].astro` — remove inline `LinkItem`, `API_BASE`, `FETCH_TIMEOUT`
- `present/index.astro` — remove inline `Presentation`, `PocketBaseResponse`, `API_BASE`, `FETCH_TIMEOUT`
- `present/[id].astro` — remove inline `API_BASE`, `FETCH_TIMEOUT`
- `sync/[id].astro` — remove inline `API_BASE`
- `sync/[id]/control.astro` — remove inline `API_BASE`, `FETCH_TIMEOUT`
- `stats.astro` — remove inline `API_BASE` if present
- `watch/[id].astro` — remove inline `API_BASE` if present

### Remove duplicate type definitions from components
- `SearchInterface.astro` script block: remove `LinkItem`, `SearchResponse`, `SearchState` interface declarations (lines 433-457) — use imports from `types/api.ts`
- `StatsSummary.astro`: remove `StatsData`, `StatsState` exports — re-export from `types/api.ts`

### Fix `control.astro` variable declaration order
- Line 19 uses `error` (`error = 'missing_token'`) before `let error = null` on line 28
- Move the `let error = null` declaration before the token check

## Acceptance Criteria

- Single source of truth for `API_BASE`, `FETCH_TIMEOUT`, and all shared interfaces
- No duplicate `LinkItem` definitions (grep for `interface LinkItem` returns only `types/api.ts`)
- All SSR pages use `apiFetch` or import `API_BASE`/`FETCH_TIMEOUT` from `lib/api.ts`
- `cd frontend && npm run build` succeeds
- No TypeScript errors
- `control.astro` no longer uses `error` before declaration
