## Context

After T-010-01 removes dead code, several patterns remain duplicated or inconsistent across Go route handlers: token generation, HTTP status codes as raw integers, missing error checks, and a leaked `admin_token` in API responses.

## Changes

### Create `routes/tokens.go` with shared token utilities
- Single `tokenLength = 32` constant (replacing `TokenLength` in `presentations.go` and `syncTokenLength` in `sync_sessions.go`)
- Shared `generateToken() (string, error)` function (replacing `generateAdminToken()` and `generateSyncAdminToken()`)
- Shared `validateToken(provided, stored string) bool` function (replacing `validateSyncToken()`)
- Update `sync_sessions.go` and `presentations.go` to use the shared functions
- Remove the old per-file duplicates

### Fix inconsistent HTTP status codes
- Replace raw integers (`500`, `200`, `400`, `404`) with `http.Status*` constants in:
  - `stats.go`: `e.JSON(500, ...)` → `e.JSON(http.StatusInternalServerError, ...)`, `e.JSON(200, ...)` → `e.JSON(http.StatusOK, ...)`
  - `links_view.go`: same pattern for `400`, `500`, `404`, `200`

### Fix `links_search.go` error handling
- Add `rows.Err()` check after iteration in `fetchTagsForLinks()` (already done — verify)
- Add actual logging at line 88-93 where comment says "Log error but don't fail the request" — use `app.Logger().Error("Failed to fetch tags", "error", err)`

### Fix `links_view.go` error handling
- Handle the swallowed `RowsAffected()` error: `rowsAffected, _ := result.RowsAffected()` → check error and log

### Remove `admin_token` from progress update response
- In `sync_sessions.go` `handleUpdateProgress()`, the response map (line 140-146) includes `"admin_token"` — remove it from the response

### Extract magic number in `stats.go`
- `LIMIT 5` appears in `sqlMostViewed` and `sqlTopTags` — extract `statsTopN = 5` constant and use `fmt.Sprintf` or keep as-is if the SQL is a const string (document the constant near the queries)

## Acceptance Criteria

- Single `routes/tokens.go` file with shared token generation and validation
- No duplicate token functions remain in `presentations.go` or `sync_sessions.go`
- No raw integer HTTP status codes in `stats.go` or `links_view.go`
- `admin_token` not present in progress update response body
- `go vet ./...` passes
- `go build .` compiles without errors
- Existing tests pass
