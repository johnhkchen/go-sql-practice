## Context

Custom Go routes for presentation lifecycle: CRUD plus a "go live" action that creates a sync session and links it to the presentation. PocketBase handles basic CRUD via its built-in API, so the custom routes focus on the live session workflow.

## Acceptance Criteria

- `POST /api/presentations/:id/live` — starts a live session:
  - Creates a new `sync_session` record (progress 0, generates admin_token)
  - Sets the presentation's `active_session` to the new session
  - Returns `{ session_id, admin_url, viewer_url, step_count, step_labels }`
- `POST /api/presentations/:id/stop` — ends the live session:
  - Clears the presentation's `active_session` field
  - Returns 200 with confirmation
- `GET /api/presentations/:id/status` — returns presentation info with live session state:
  - Includes `is_live`, current `progress`, `current_step` (computed from progress and step_count), `viewer_count` (if available from PocketBase, else omit)
- Step-to-progress mapping: `step_progress = step_index / (step_count - 1)` for step_count > 1, or `0.0` for single-step
- All endpoints return 404 if presentation doesn't exist
- CRUD for presentations uses PocketBase's built-in collection API (no custom routes needed)
