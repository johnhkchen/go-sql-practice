## Context

Create a Makefile (or Taskfile) that orchestrates the full build pipeline: install frontend dependencies, build the Astro frontend, then compile the Go binary with the embedded assets. Having a single `make build` command eliminates "works on my machine" issues.

## Acceptance Criteria

- `make build` (or equivalent) runs the full pipeline: frontend install, frontend build, go build
- `make dev` starts PocketBase in dev mode (with hot reload if feasible, or just a restart-on-change watcher)
- `make test` runs Go tests
- `make clean` removes build artifacts (binary, frontend/dist, pb_data if present)
- Each target can be run independently
- Makefile uses phony targets and has brief comments explaining each target
