## Context

`routes/presentations.go` and `routes/sync_sessions.go` use `e.Request.PathValue("id")` to extract route parameters. This is a Go 1.22+ stdlib method that may not work with PocketBase v0.36.5's router (which uses Echo). If `PathValue()` returns empty strings, all presentation and sync endpoints silently fail.

## Acceptance Criteria

- Verify whether `e.Request.PathValue("id")` works with PocketBase v0.36.5's routing
- If it doesn't, switch to PocketBase/Echo's parameter extraction (e.g., `e.Request.URL.Query()` or route group patterns)
- All presentation endpoints work: `/api/presentations/:id/live`, `/stop`, `/status`
- All sync endpoints work: `/api/sync/:id/progress`
- Add a simple test or verification that route parameters are correctly extracted
