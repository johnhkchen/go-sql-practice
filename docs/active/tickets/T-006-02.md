## Context

Custom Go routes for creating and controlling sync sessions. The create endpoint generates a session with a random admin token and returns both the admin and viewer URLs. The progress endpoint lets the admin push a new progress value (which PocketBase's realtime then broadcasts to all subscribers).

## Acceptance Criteria

- `POST /api/sync/create` creates a new sync_session record and returns:
  ```json
  {
    "session_id": "abc123",
    "admin_url": "/sync/abc123/control?token=<admin_token>",
    "viewer_url": "/sync/abc123"
  }
  ```
- `POST /api/sync/:id/progress` with JSON body `{"progress": 0.42}` and query param `?token=<admin_token>`:
  - Validates token matches the session's `admin_token`
  - Validates progress is a float between 0 and 1 (inclusive)
  - Updates the session record's `progress` field
  - Returns 200 with the updated session
  - Returns 403 if token is wrong, 404 if session doesn't exist, 400 if progress is out of range
- Admin token is a cryptographically random string (e.g., 32 hex chars)
- No authentication beyond the admin token — viewer endpoints are fully public
