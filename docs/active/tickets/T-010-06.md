## Context

Six `.astro` files contain large `<script>` blocks with full class definitions (100-500+ lines each). These inline scripts are hard to test, lint, and reuse. Extracting them to standalone `.ts` files improves editor support, enables future unit testing, and keeps Astro components focused on markup.

## Changes

### Create `frontend/src/scripts/` directory

### Extract `SyncViewer` class
- **From:** `sync/[id].astro` `<script>` block (~160 lines starting at line 136)
- **To:** `frontend/src/scripts/syncViewer.ts`
- Export the `SyncViewer` class
- The `.astro` file imports and instantiates it in a minimal `<script>` tag

### Extract `SyncController` class
- **From:** `sync/[id]/control.astro` `<script>` block (~350 lines starting at line 204)
- **To:** `frontend/src/scripts/syncController.ts`
- Export the `SyncController` class

### Extract `PresentationAutoViewer` class
- **From:** `watch/[id].astro` `<script>` block (~350 lines starting at line 598)
- **To:** `frontend/src/scripts/presentationViewer.ts`
- Export the `PresentationAutoViewer` class
- Import `stepConversion` utility from shared types instead of inlining

### Extract `PresenterController` class
- **From:** `PresenterController.astro` `<script>` block (~200 lines starting at line 804)
- **To:** `frontend/src/scripts/presenterController.ts`
- Export the `PresenterController` class

### Extract `StatsController` class
- **From:** `StatsSummary.astro` `<script>` block (~270 lines starting at line 110)
- **To:** `frontend/src/scripts/statsController.ts`
- Export the `StatsController` class
- Import `StatsData`, `StatsState` from `types/api.ts` (created in T-010-03)

### Extract search enhancement
- **From:** `SearchInterface.astro` `<script>` block (~270 lines starting at line 431)
- **To:** `frontend/src/scripts/searchEnhancer.ts`
- Import `LinkItem`, `SearchResponse` from `types/api.ts`
- Fix the client-side search behavior: currently falls back to `window.location.reload()` for non-empty results (line 657) — either implement proper DOM update with template rendering, or simplify to always use server-side navigation (removing the incomplete client-side path)

### Pattern for each extraction
Each `.astro` file should have a minimal `<script>` block like:
```html
<script>
  import { SyncViewer } from '../scripts/syncViewer';
  document.querySelectorAll('.sync-viewer').forEach(el => {
    new SyncViewer(el as HTMLElement);
  });
</script>
```

## Acceptance Criteria

- 6 new `.ts` files in `frontend/src/scripts/`
- Each `.astro` file's `<script>` block is reduced to < 10 lines (import + instantiation)
- All extracted classes maintain their existing functionality
- `SearchInterface` client-side search either properly updates DOM or cleanly delegates to server navigation (no half-working JS enhancement)
- `cd frontend && npm run build` succeeds
- No TypeScript errors
