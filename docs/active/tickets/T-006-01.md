## Context

Create a PocketBase collection for sync sessions. Each session holds a progress float and an admin token. PocketBase's built-in realtime subscriptions will handle broadcasting changes to viewers — no custom pub/sub needed.

## Acceptance Criteria

- Migration creates a `sync_sessions` collection with fields:
  - `progress` (number, min 0, max 1, default 0)
  - `admin_token` (text, required) — opaque token for admin access, generated server-side
- Collection has API rules that allow:
  - Anyone to **view** a session record (viewer access)
  - Only requests with a matching `admin_token` to **update** (checked via custom route, not PocketBase rules)
- PocketBase realtime subscriptions work on this collection (default behavior, just verify it's not disabled)
- Starting the app creates the collection alongside existing ones
